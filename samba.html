Simple Samba Server


Simple Samba Server. What is it? Do use a thumb drive or USB hard drive to move files between computers. Running a Samba program on your Rasberry Pi will give you a shared directory, to easily move those files. This config file is easy and has no built in security, other than it only allows computers in your house to access the shared drive. Size of the drive is limited to amount of memory available, so if you want to have gobs of memory, you will need to use an external USB drive plugged into your RPi.

First you will have to load samba, so
$sudo apt install samba

Before editing your smb.conf file, the way I did
$cd /etc/samba
$sudo mv smb.conf smb.conf.org
to save the orignal

$sudo touch smb.conf

now edit your smb.conf file to look like this
----------------------------------------------------------
[global]
    workgroup = WORKGROUP
    hosts allow = 192.168.2.
    hosts deny = ALL
    log file = /var/log/samba/log.%m
    logging = file
    map to guest = bad user

[Shares]
    path = /home/Shared
    guest ok = yes
    read only = no
    create mask = 0777
    directory mask = 0777
    force user = nobody

----------------------------------------------------------

Be sure and create the mount point for your share, with
$sudo mkdir /home/Shared

After creating your directory, give everyone rights
$sudo chmod 777 /home/Shared
$sudo chown -R nobody.nogroup /home/Shared

To test config file, use;
$sudo testparm

After any changes to the smb.conf file, use this to restart
$sudo service smbd restart

At this point, you should find the computer and the share under network on your computer.
You may also wish to rt click on Shared and map a drive letter.
